## 如何查看工具调用日志

在君君的日志中，搜索以下关键词：
- `[tool_use]` - 工具调用模块的日志
- `开始执行工具调用` - 工具开始执行
- `web_search` - 搜索工具被调用
- `工具执行完成` - 工具执行结果

## 如果搜索不工作

### 检查清单：
1. ✅ 工具功能已启用（`enable_tool = true`）
2. ✅ 搜索插件已启用（`plugins/google_search_plugin/config.toml` 中 `enabled = true`）
3. ✅ 至少一个搜索引擎已启用（Bing、搜狗、DuckDuckGo）
4. ⚠️ 检查日志中是否有 `[tool_use]` 相关输出

### 调试步骤：
1. 重启君君和 NapCat 适配器
2. 发送测试消息："帮我查一下今天的新闻"
3. 查看日志，搜索 `[tool_use]`
4. 如果没有 `[tool_use]` 日志，说明工具调用模块没有被触发
5. 如果有 `[tool_use]` 但显示"无需执行工具"，说明模型认为不需要搜索

## 启用详细日志（可选）

如果需要更详细的调试信息，可以在 `bot_config.toml` 中启用：

```toml
[debug]
show_prompt = true  # 显示所有 prompt
show_replyer_prompt = true  # 显示回复器 prompt
```

重启后会输出更详细的日志。

## 注意事项

- 搜索功能依赖网络连接
- Bing、搜狗、DuckDuckGo 不需要 API key
- Google 和 Tavily 需要 API key（目前未启用）
- 搜索结果会被缓存 3 个周期，避免重复搜索
