## ✨ 新架构的优势

### 1. 工具调用时机正确

**旧架构问题：**
```
用户："saber线重置是真的吗？"
君君："我搜搜看~"（但实际没搜索）
```

**新架构解决：**
```
用户："saber线重置是真的吗？"
君君：（先搜索）→（基于搜索结果回复）"根据最新消息..."
```

### 2. 减少 LLM 调用次数

| 架构 | LLM 调用次数 | 说明 |
|------|-------------|------|
| 旧架构 | 3 次 | Planner + ToolUse + Replyer |
| 新架构 | 1-2 次 | 决策+工具调用 + 最终回复（仅在需要工具时） |

**成本降低：33% - 66%**

### 3. 简化的处理流程

**旧架构（7 层）：**
```
消息接收 → 预处理 → 心流 → 大脑循环 → Planner → Replyer → ToolUse → 发送
```

**新架构（3 层）：**
```
消息接收 → UnifiedAgent → 发送
```

### 4. 并行工具执行

新架构支持多个工具并行执行，大幅降低延迟：

```python
# 旧架构：串行执行
工具1 (5s) → 工具2 (5s) → 工具3 (5s) = 15s

# 新架构：并行执行
工具1 (5s) ┐
工具2 (5s) ├→ 最长 5s
工具3 (5s) ┘
```

## 📝 新架构核心代码说明

### UnifiedAgent 工作流程

```python
async def process(message):
    # 1. 构建上下文
    context = await _build_context(message)

    # 2. LLM 决策（带工具定义）
    context = await _llm_decision(context)
    # → LLM 自己决定是否需要工具

    # 3. 如果需要工具，并行执行
    if context.need_tools:
        context = await _execute_tools(context)

        # 4. 基于工具结果生成最终回复
        context = await _llm_final_reply(context)

    return context.final_response
```

### 工具执行器

```python
async def execute_tools(tool_calls):
    # 并行执行所有工具
    tasks = [execute_single_tool(tc) for tc in tool_calls]
    results = await asyncio.gather(*tasks)
    return results
```
