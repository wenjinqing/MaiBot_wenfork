# 多机器人端口配置指南

## 问题说明

在多机器人模式下，每个机器人实例都需要独立的 `maim_message` 服务器端口。如果多个机器人使用相同的端口，会导致端口冲突，机器人无法正常启动。

## 端口分配方案

### 默认端口分配

| 服务 | 端口 | 配置位置 | 说明 |
|------|------|----------|------|
| 主程序 | 8000 | `.env` 中的 `PORT` | MaiBot 主程序监听端口（已弃用） |
| WebUI | 8001 | `.env` 中的 `WEBUI_PORT` | Web 管理界面端口 |
| 机器人1（君君）消息服务器 | 8091 | `config/bot_config.toml` 的 `[maim_message]` | 君君的消息服务器端口 |
| 机器人1（君君）NapCat | 8095 | NapCat 配置 | 君君的 NapCat 服务端口 |
| 机器人2（伊伊）消息服务器 | 8092 | `config/yiyi_bot_config.toml` 的 `[maim_message]` | 伊伊的消息服务器端口 |
| 机器人2（伊伊）NapCat | 8097 | NapCat 配置 | 伊伊的 NapCat 服务端口 |
| 机器人3 消息服务器 | 8093 | `config/bot3_config.toml` | 第三个机器人的消息服务器 |
| 机器人3 NapCat | 8098 | NapCat 配置 | 第三个机器人的 NapCat 服务端口 |
| ... | 8094+ | ... | 更多机器人依次递增 |

## 配置步骤

### 1. 修改机器人配置文件

每个机器人的配置文件（如 `config/bot_config.toml`）中，找到 `[maim_message]` 部分：

```toml
[maim_message]
auth_token = []
use_custom = true  # 必须设置为 true，启用自定义端口
host = "127.0.0.1"
port = 8091  # 每个机器人使用不同的端口
mode = "ws"
use_wss = false
cert_file = ""
key_file = ""
```

### 2. 为每个机器人分配唯一端口

**机器人1（君君）- `config/bot_config.toml`：**
```toml
[maim_message]
use_custom = true
port = 8091
```

**机器人2（伊伊）- `config/yiyi_bot_config.toml`：**
```toml
[maim_message]
use_custom = true
port = 8092
```

**机器人3 - `config/bot3_config.toml`：**
```toml
[maim_message]
use_custom = true
port = 8093
```

### 3. 配置 NapCat 适配器

**关键：** 每个机器人需要独立的 NapCat 适配器实例，并配置正确的端口连接。

#### 君君的适配器配置

创建 `MaiBot-Napcat-Adapter/config.toml`：

```toml
[napcat_server]
host = "localhost"
port = 8095  # 君君的 NapCat 服务端口

[maibot_server]
host = "localhost"
port = 8091  # 连接到君君的消息服务器端口
```

#### 伊伊的适配器配置

创建 `MaiBot-Napcat-Adapter-Yiyi/config.toml`：

```toml
[napcat_server]
host = "localhost"
port = 8097  # 伊伊的 NapCat 服务端口

[maibot_server]
host = "localhost"
port = 8092  # 连接到伊伊的消息服务器端口
```

#### 启动适配器

```bash
# 启动君君的适配器
cd MaiBot-Napcat-Adapter
python main.py

# 在另一个终端启动伊伊的适配器
cd MaiBot-Napcat-Adapter-Yiyi
python main.py
```

## 常见问题

### Q1: 端口被占用怎么办？

**错误信息：**
```
OSError: [Errno 48] Address already in use
```

**解决方法：**
1. 检查是否有其他程序占用了该端口
2. 使用命令查看端口占用情况：
   ```bash
   # Windows
   netstat -ano | findstr :8091

   # Linux/macOS
   lsof -i :8091
   ```
3. 更换为其他未被占用的端口

### Q2: 多个机器人使用相同端口会怎样？

会导致第二个机器人启动失败，出现端口冲突错误。只有第一个启动的机器人能正常工作。

### Q3: 可以使用其他端口范围吗？

可以。端口号范围是 1024-65535，建议使用 8000-9000 之间的端口，避免与常用服务冲突。

### Q4: use_custom 必须设置为 true 吗？

在多机器人模式下，**必须**将 `use_custom` 设置为 `true`。如果设置为 `false`，所有机器人会尝试使用默认端口 8090，导致冲突。

## 端口规划建议

### 小型部署（2-3个机器人）
- 8091, 8092, 8093

### 中型部署（4-10个机器人）
- 8091-8100

### 大型部署（10+个机器人）
- 考虑使用不同的端口段，如：
  - 第一组：8091-8099
  - 第二组：8101-8109
  - 第三组：8201-8209

## 验证配置

启动 MaiBot 后，检查日志输出：

```
[INFO] 正在初始化机器人: junjun_main (君君)
[INFO] maim_message 服务器启动在 ws://127.0.0.1:8091
[INFO] ✓ 机器人 junjun_main 初始化成功

[INFO] 正在初始化机器人: yiyi_bot (伊伊)
[INFO] maim_message 服务器启动在 ws://127.0.0.1:8092
[INFO] ✓ 机器人 yiyi_bot 初始化成功
```

如果看到类似输出，说明端口配置正确。

## 配置模板

创建新机器人时，可以使用以下模板：

```toml
[inner]
version = "6.23.5"

[bot]
platform = "qq"
qq_account = "${YOUR_BOT_QQ_ACCOUNT}"
nickname = "你的机器人昵称"

# ... 其他配置 ...

[maim_message]
auth_token = []
use_custom = true
host = "127.0.0.1"
port = 8093  # 修改为未使用的端口
mode = "ws"
use_wss = false
cert_file = ""
key_file = ""
```
