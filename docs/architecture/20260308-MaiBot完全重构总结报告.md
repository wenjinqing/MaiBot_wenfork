3. ✅ **LLM 调用次数过多** - 从 3 次减少到 1-2 次
4. ✅ **模块耦合度高** - 采用清晰的接口和职责分离
5. ✅ **缺乏并行执行** - 实现工具并行执行

## 🏗️ 架构设计

### 旧架构 vs 新架构

#### 旧架构（7 层）
```
消息接收 (MessageServer)
    ↓
消息路由 (MessageRouter)
    ↓
预处理 (ChatBot.message_process)
    ↓
心流处理 (HeartFCMessageReceiver)
    ↓
大脑循环 (BrainChatting)
    ↓
动作规划 (ActionPlanner) → LLM 调用 1
    ↓
回复生成 (Replyer) → LLM 调用 2
    ↓
工具执行 (ToolUse) → LLM 调用 3 ❌ 太晚了！
    ↓
消息发送
```

#### 新架构（3 层）
```
消息接收
    ↓
UnifiedAgent
    ├─ LLM 决策（带工具定义）→ LLM 调用 1
    ├─ 并行执行工具 ✅ 在回复前执行！
    └─ 基于工具结果生成回复 → LLM 调用 2（仅在需要时）
    ↓
消息发送
```

## 🔧 核心组件详解

### 1. UnifiedChatAgent（统一聊天 Agent）

**文件：** `src/chat_v2/agent/unified_agent.py`

**职责：**
- 统一处理消息的决策、工具调用、回复生成
- 使用 Function Calling 模式，让 LLM 自己决定是否需要工具

**核心流程：**
```python
async def process(message):
    # 1. 构建上下文
    context = await _build_context(message)

    # 2. LLM 决策（带工具定义）
    context = await _llm_decision(context)
    # → LLM 返回：是否需要工具 + 工具调用列表

    # 3. 如果需要工具，并行执行
    if context.need_tools:
        context = await _execute_tools(context)

        # 4. 基于工具结果生成最终回复
        context = await _llm_final_reply(context)

    return ExecutionResult(
        success=True,
        response=context.final_response,
        llm_calls=context.llm_calls,
        tool_calls=len(context.tool_results)
    )
```

**关键特性：**
- ✅ 工具在回复前执行
- ✅ 支持 Function Calling
- ✅ 自动判断是否需要工具
- ✅ 完整的执行上下文追踪

### 2. ToolExecutor（并行工具执行器）

**文件：** `src/chat_v2/executor/tool_executor.py`

**职责：**
- 并行执行多个工具��用
- 统一的超时和错误处理
- 格式化工具结果

**核心代码：**
```python
async def execute_tools(tool_calls, timeout=30.0):
    # 创建并行任务
    tasks = [
        _execute_single_tool(tc, timeout)
        for tc in tool_calls
    ]

    # 并行执行
    results = await asyncio.gather(*tasks, return_exceptions=True)

    return [处理后的结果]
```

**性能提升：**
```
串行执行（旧）：工具1(5s) + 工具2(5s) + 工具3(5s) = 15s
并行执行（新）：max(5s, 5s, 5s) = 5s
提升：3倍
```

### 3. MessageHandler（消息处理器）

**文件：** `src/chat_v2/handler/message_handler.py`

**职责：**
- 管理 Agent 实例（每个聊天一个 Agent）
- 统一的消息处理入口
- 统计和日志记录

**核心代码：**
```python
async def handle_message(message, chat_stream):
    # 获取或创建 Agent
    if chat_id not in self.agents:
        self.agents[chat_id] = UnifiedChatAgent(chat_stream)

    agent = self.agents[chat_id]

    # 处理消息
    result = await agent.process(message)

    return result.response
```

### 4. 数据模型

**文件：** `src/chat_v2/models/context.py`

#### AgentContext
```python
@dataclass
class AgentContext:
    message: Any                    # 当前消息
    chat_history: List[Any]         # 聊天历史
```
