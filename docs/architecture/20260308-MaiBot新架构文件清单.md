## 🎯 核心文件说明

### 1. unified_agent.py（最重要）

**路径：** `src/chat_v2/agent/unified_agent.py`

**职责：**
- 统一的消息处理流程
- LLM 决策和工具调用
- 基于工具结果生成回复

**核心方法：**
- `process()` - 主处理流程
- `_build_context()` - 构建上下文
- `_llm_decision()` - LLM 决策
- `_execute_tools()` - 执行工具
- `_llm_final_reply()` - 生成最终回复

### 2. tool_executor.py

**路径：** `src/chat_v2/executor/tool_executor.py`

**职责：**
- 并行执行工具
- 超时和错误处理
- 结果格式化

**核心方法：**
- `execute_tools()` - 并行执行
- `_execute_single_tool()` - 单个工具执行
- `_format_tool_result()` - 格式化结果

### 3. message_handler.py

**路径：** `src/chat_v2/handler/message_handler.py`

**职责：**
- Agent 实例管理
- 消息路由
- 统计和日志

**核心方法：**
- `handle_message()` - 处理消息
- `clear_agent()` - 清除 Agent
- `get_message_handler()` - 获取单例

### 4. context.py

**路径：** `src/chat_v2/models/context.py`

**职责：**
- 定义核心数据结构
- 执行状态追踪

**核心类：**
- `AgentContext` - Agent 上下文
- `ExecutionResult` - 执行结果
- `ToolCall` - 工具调用
- `ToolResult` - 工具结果
- `ExecutionStatus` - 执行状态枚举

## 🔧 配置文件

### bot_config.toml

**修改内容：**

```toml
[inner]
version = "6.23.5"

# 新架构开关（实验性功能）
use_v2_architecture = false  # true=新架构, false=旧架构
```

**说明：**
- 默认为 `false`，使用旧架构
- 改为 `true` 启用新架构
- 可随时切换，无需修改代码
