# MaiBot 重构方案 A：最小改动方案

## 目标
在不破坏现有架构的前提下，修复工具调用时机问题，让君君能够真正使用搜索功能。

## 核心改动

### 1. 将 ToolUse 提前到 Planner 之后

**当前流程：**
```
Planner 决策 → Replyer 生成回复 → ToolUse 执行工具（太晚了！）
```

**改进流程：**
```
Planner 决策 → ToolUse 执行工具 → Replyer 基于工具结果生成回复
```

### 2. 修改文件

#### 文件 1: `src/chat/brain_chat/brain_chat.py`

**位置：** `_main_chat_loop()` 方法，第 545-560 行

**当前代码：**
```python
elif action_planner_info.action_type == "reply":
    try:
        success, llm_response = await generator_api.generate_reply(
            chat_stream=self.chat_stream,
            reply_message=action_planner_info.action_message,
            available_actions=available_actions,
            chosen_actions=chosen_action_plan_infos,
            reply_reason=action_planner_info.reasoning or "",
            enable_tool=global_config.tool.enable_tool,  # 工具在这里被调用
            request_type="replyer",
            from_plugin=False,
        )
```

**修改为：**
```python
elif action_planner_info.action_type == "reply":
    try:
        # 1. 先执行工具调用（如果需要）
        tool_context = ""
        if global_config.tool.enable_tool:
            from src.plugin_system.core.tool_use import ToolExecutor

            # 获取最近的聊天历史
            chat_history = await self.chat_stream.get_raw_msg_before_timestamp_with_chat(
                timestamp=time.time(),
                limit=10
            )

            # 执行工具调用
            tool_executor = ToolExecutor(chat_id=self.chat_stream.stream_id)
            tool_results, used_tools, _ = await tool_executor.execute_from_chat_message(
                target_message=action_planner_info.action_message.content if action_planner_info.action_message else "",
                chat_history=chat_history,
                sender=action_planner_info.action_message.sender if action_planner_info.action_message else "",
                return_details=True
            )

            # 格式化工具结果为上下文
            if tool_results:
                tool_context = "以下是你通过工具获取到的实时信息：\n"
                for tool_result in tool_results:
                    tool_name = tool_result.get("tool_name", "unknown")
                    content = tool_result.get("content", "")
                    tool_context += f"- 【{tool_name}】: {content}\n"
                tool_context += "\n请基于以上信息回复用户。\n"

                logger.info(f"工具执行完成，共执行 {len(used_tools)} 个工具: {used_tools}")

        # 2. 生成回复（基于工具结果）
        success, llm_response = await generator_api.generate_reply(
            chat_stream=self.chat_stream,
            reply_message=action_planner_info.action_message,
            available_actions=available_actions,
            chosen_actions=chosen_action_plan_infos,
            reply_reason=action_planner_info.reasoning or "",
            extra_info=tool_context,  # 将工具结果作为额外信息传入
            enable_tool=False,  # 禁用 Replyer 内部的工具调用
            request_type="replyer",
            from_plugin=False,
        )
```

#### 文件 2: `src/chat/replyer/group_generator.py`

---

## 优点

1. ✅ **改动最小**：只修改一个文件的一个方法
2. ✅ **风险最低**：不破坏现有架构
3. ✅ **立即生效**：修改后重启即可
4. ✅ **解决核心问题**：工具在回复前执行，君君能真正搜索

## 缺点

1. ⚠️ 仍然有 2 次 LLM 调用（Planner + ToolUse）
2. ⚠️ 没有解决架构耦合问题
3. ⚠️ 没有优化性能
