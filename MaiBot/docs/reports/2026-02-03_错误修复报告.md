## 问题描述

用户运行 `bot.py` 时遇到以下错误：

```
ModuleNotFoundError: No module named 'maim_message'
```

## 根本原因

有两个问题：

### 1. 虚拟环境未激活
- 用户使用的是系统 Python (`D:\python38`)
- 而项目依赖安装在虚拟环境 `.venv` 中
- `maim-message` 包已安装在 `.venv` 中，但系统 Python 无法访问

### 2. 新异常模块缺少向后兼容性
- 我们创建了新的异常体系（任务9），但旧代码仍在使用旧的异常类名
- 缺少的异常类：
  - `RespParseException`
  - `RespNotOkException`
  - `ReqAbortException`
  - `EmptyResponseException`

---

## 解决方案

### 1. 虚拟环境问题

**解决方法**: 使用虚拟环境中的 Python 运行程序

```bash
# Windows
cd E:\MaiM\MaiM-with-u\MaiBot
.venv\Scripts\python.exe bot.py

# 或激活虚拟环境
.venv\Scripts\activate
python bot.py
```

### 2. 异常兼容性问题

**解决方法**: 在新异常模块中添加向后兼容的包装类

**修改文件**: `src/llm_models/exceptions/__init__.py`

**添加的代码**:

```python
# 向后兼容：为旧代码提供包装类
class RespNotOkException(APIResponseError):
    """响应异常（向后兼容）"""
    def __init__(self, status_code: int, message: str = None):
        self.status_code = status_code
        super().__init__(
            message=message or f"HTTP {status_code}",
            error_code=f"HTTP_{status_code}",
            recoverable=status_code in [429, 500, 503],
            retry_after=60 if status_code == 429 else None,
            details={"status_code": status_code}
        )

class RespParseException(APIParseError):
    """响应解析异常（向后兼容）"""
    def __init__(self, ext_info, message: str = None):
        self.ext_info = ext_info
        super().__init__(
            message=message or "响应解析失败",
            error_code="PARSE_ERROR",
            recoverable=False,
            details={"ext_info": ext_info}
        )

class ReqAbortException(NetworkException):
    """请求中止异常（向后兼容）"""
    def __init__(self, message: str = None):
        super().__init__(
            message=message or "请求因未知原因异常终止",
            error_code="REQUEST_ABORTED",
            recoverable=True,
            retry_after=5
        )

class EmptyResponseException(APIResponseError):
    """空响应异常（向后兼容）"""
    def __init__(self, message: str = "响应内容为空，这可能是一个临时性问题"):
        super().__init__(
            message=message,
            error_code="EMPTY_RESPONSE",
            recoverable=True,
            retry_after=5
        )
```

我们的新异常体系保持了与旧代码的完全兼容：

- **新代码**: 可以使用新的异常类（`APIParseError`, `APIResponseError` 等）
