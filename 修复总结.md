### 问题1：60秒新闻调用太频繁 ✓ 已修复

**原因：**
- `get_60s_news` 工具的 `available_for_llm` 设置为 `True`
- 描述太宽泛："用户问新闻/时事时调用"，导致 LLM 在任何情况下都会调用

**修复方案：**
- 将 `available_for_llm` 改为 `False`
- 修改描述为："仅当用户明确要求'新闻'、'今日新闻'、'热点'时调用，不要在其他情况下使用"
- 已同步到君君和伊伊

## 技术细节

### InnerConfig 类定义
```python
@dataclass
class InnerConfig(ConfigBase):
    """内部配置类 - 用于版本管理和实验性功能"""

    version: str = field(default="0.0.0")
    """配置文件版本号"""

    use_v2_architecture: bool = False
    """是否使用新的 chat_v2 架构（实验性功能）"""
```

### Config 类添加字段
```python
# 内部配置
inner: InnerConfig = field(default_factory=InnerConfig)
"""内部配置（版本管理和实验性功能）"""
```
