# 插件开发指南

## 简介

MaiBot 的插件系统提供了灵活的扩展机制，允许开发者通过插件的方式为机器人添加新功能。本指南将帮助你快速上手插件开发。

## 插件类型

MaiBot 支持三种主要的插件组件类型：

### 1. Command（命令）

命令是用户通过特定格式触发的功能，通常以 `/` 开头。

**特点：**
- 明确的触发条件（正则匹配）
- 可以拦截消息，阻止后续处理
- 适合工具类功能

**示例：**
```python
from src.plugin_system.base import BaseCommand

class HelloCommand(BaseCommand):
    command_name = "hello"
    command_description = "打招呼命令"
    command_pattern = r"^/hello(?:\s+(.+))?$"
    command_help = "用法：/hello [名字]"
    intercept_message = True  # 拦截消息

    async def execute(self):
        # 获取匹配的参数
        name = self.matched_groups[0] if self.matched_groups else "世界"

        # 发送回复
        await self.send_text(f"你好，{name}！")

        return True, "Success", True
```

### 2. Action（动作）

动作是机器人在对话过程中可以执行的行为，由思维流系统调度。

**特点：**
- 灵活的激活条件（关键词、LLM 判断、随机等）
- 不拦截消息，可与其他动作并行
- 适合增强对话能力

**激活类型：**
- `ALWAYS` - 总是激活
- `KEYWORD` - 关键词触发
- `LLM_JUDGE` - LLM 判断是否激活
- `RANDOM` - 随机激活

**示例：**
```python
from src.plugin_system.base import BaseAction, ActionActivationType
from src.plugin_system.base.component_types import ChatMode

class ImageAction(BaseAction):
    action_name = "send_image"
    action_description = "发送图片"
    activation_type = ActionActivationType.KEYWORD
    activation_keywords = ["看看图片", "发图"]
    keyword_case_sensitive = False
    mode_enable = ChatMode.ALL  # 所有模式启用

    async def execute(self):
        # 获取图片
        image_url = "https://example.com/image.jpg"

        # 发送图片
        await self.send_image(image_url)

        return True, "图片发送成功"
```

### 3. Event（事件）

事件监听器可以在特定事件发生时执行逻辑。

**支持的事件类型：**
- `ON_MESSAGE_PRE_PROCESS` - 消息预处理
- `ON_MESSAGE` - 消息处理
- `POST_SEND_PRE_PROCESS` - 发送前预处理
- `POST_SEND` - 发送后处理

**示例：**
```python
from src.plugin_system.base import BaseEvent, EventType

class MessageLogEvent(BaseEvent):
    event_name = "message_logger"
    event_type = EventType.ON_MESSAGE

    async def handle(self, message, **kwargs):
        # 记录消息
        print(f"收到消息: {message.processed_plain_text}")

        # 返回 True 继续处理，False 中断
        return True, message
```

## 插件结构

### 标准插件目录结构

```
my_plugin/
├── __init__.py          # 插件包初始化
├── plugin.py            # 插件主文件
├── config.toml          # 插件配置
├── _manifest.json       # 插件元信息
├── commands.py          # 命令实现（可选）
├── actions.py           # 动作实现（可选）
└── events.py            # 事件实现（可选）
```

### 插件主文件 (plugin.py)

```python
from typing import List, Tuple, Type
from src.plugin_system.base import BasePlugin, register_plugin
from src.plugin_system.base import ComponentInfo

from .commands import HelloCommand
from .actions import ImageAction

@register_plugin
class MyPlugin(BasePlugin):
    """我的插件"""

    plugin_name: str = "my_plugin"
    enable_plugin: bool = True

    def get_plugin_components(self) -> List[Tuple[ComponentInfo, Type]]:
        """注册插件组件"""
        components = []

        # 注册命令
        if self.get_config("components.enable_hello_command", True):
            components.append((
                HelloCommand.get_command_info(),
                HelloCommand
            ))

        # 注册动作
        if self.get_config("components.enable_image_action", True):
            components.append((
                ImageAction.get_action_info(),
                ImageAction
            ))

        return components
```

### 配置文件 (config.toml)

```toml
[plugin]
enable = true
description = "我的插件"

[components]
enable_hello_command = true
enable_image_action = true

[settings]
api_key = ""
timeout = 30
```

### 元信息文件 (_manifest.json)

```json
{
  "plugin_name": "my_plugin",
  "version": "1.0.0",
  "description": "我的第一个插件",
  "author": "Your Name",
  "dependencies": []
}
```

## 开发流程

### 1. 创建插件目录

```bash
mkdir plugins/my_plugin
cd plugins/my_plugin
```

### 2. 创建必要文件

```bash
touch __init__.py plugin.py config.toml _manifest.json
```

### 3. 实现插件逻辑

```bash
tail -f logs/app_*.log.jsonl
```

## API 参考

### BaseCommand 常用方法

```python
# 发送文本消息
await self.send_text("Hello")

# 发送图片
await self.send_image(image_base64)

# 发送表情包
await self.send_emoji(emoji_base64)

# 发送混合消息（文字+图片）
await self.send_hybrid([
    (ReplyContentType.TEXT, "文字"),
    (ReplyContentType.IMAGE, image_url)
])

# 获取匹配的参数
groups = self.matched_groups
```

### BaseAction 常用方法

```python
# 发送消息（同 BaseCommand）
await self.send_text("Hello")

# 获取聊天信息
chat_id = self.chat_id
user_id = self.user_id
is_group = self.is_group

# 获取插件配置
api_key = self.plugin_config.get("api_key")
```

### 配置访问

```python
# 在插件类中
value = self.get_config("settings.api_key", "default_value")

# 在组件中
value = self.plugin_config.get("api_key")
```

## 最佳实践

### 1. 错误处理

```python
async def execute(self):
    try:
        # 你的逻辑
        result = await some_async_function()
        return True, "Success"
    except Exception as e:
        logger.error(f"执行失败: {e}")
        await self.send_text(f"❌ 操作失败: {str(e)}")
        return False, str(e)
```

## 常见问题

### Q: 插件没有被加载？

A: 检查以下几点：
1. 插件目录是否在 `plugins/` 下
2. `plugin.py` 中是否使用了 `@register_plugin` 装饰器
3. `config.toml` 中 `enable = true`
4. 查看启动日志是否有错误信息

### Q: 命令没有响应？

A: 检查：
1. `command_pattern` 正则表达式是否正确
2. 命令是否被其他插件拦截
3. 查看日志中的命令匹配信息

### Q: 如何调试插件？

A:
1. 使用 `logger.debug()` 输出调试信息
2. 查看 `logs/` 目录下的日志文件
3. 在 Web UI 中查看实时日志
