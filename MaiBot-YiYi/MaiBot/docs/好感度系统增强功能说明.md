# 好感度系统增强功能说明

## 新增功能

### 1. 主动对话时根据好感度 @ 群成员

**功能描述：**
- 机器人在群聊中主动发起话题时，会根据群成员的好感度加权随机选择要 @ 的对象
- 好感度越高的用户，被选中的概率越大
- 这样可以让机器人更倾向于和关系好的用户互动

**实现位置：**
- `src/proactive_system/proactive_chat_manager.py`
- 新增方法：`_select_group_member_by_intimacy()`

**工作原理：**
1. 获取该群最近发言的用户（最多 50 人）
2. 从数据库查询这些用户的好感度信息
3. 按好感度降序排列
4. 使用加权随机算法选择（权重 = 好感度 + 10）
5. 在主动对话时 @ 选中的用户

**示例：**
```
群成员好感度：
- 张三：85.0（挚友）
- 李四：60.0（好友）
- 王五：30.0（认识）

选中概率：
- 张三：95/(95+70+40) ≈ 46%
- 李四：70/(95+70+40) ≈ 34%
- 王五：40/(95+70+40) ≈ 20%
```

### 2. 好感度查询关键词触发

**功能描述：**
- 用户可以通过发送关键词查询自己的好感度信息
- 使用大模型生成自然、口语化的回复，而不是生硬的模板
- 根据关系等级（陌生/熟人/好友/挚友/恋人）调整语气和亲密程度
- 每次查询的回复都略有不同，更加自然真实

**触发关键词：**
- 好感度
- 亲密度
- 关系值
- 我们的关系
- 查询好感度
- 查询亲密度
- 查看好感度
- 查看亲密度
- 好感
- 亲密
- 关系
- 喜欢我吗
- 对我的印象

**实现位置：**
- `plugins/intimacy_query_plugin/` - 插件目录
- `src/common/intimacy_query_action.py` - 查询动作（使用 LLM 生成回复）
- `src/common/relationship_query.py` - 查询工具（已存在）

**回复特点：**

1. **自然口语化** - 不使用生硬的模板
   ```
   旧版（模板）：
   📊 关系信息查询
   👤 用户：张三
   💖 好感度：65.0/100

   新版（自然）：
   嗯...让我想想，我对你的好感度大概有 65 分左右呢~
   我们现在算是好朋友啦，聊了好多次天了，
   感觉和你聊天挺开心的喵~
   ```

2. **根据关系等级调整语气**
   - 挚友/恋人（80-100分）：更亲密、随意、可以撒娇
   - 好友（60-79分）：轻松友好、可以开玩笑
   - 熟人（40-59分）：友好但保持一定距离
   - 陌生（0-39分）：礼貌友好、鼓励多交流

3. **包含关键信息**
   - 好感度数值
   - 关系等级描述
   - 心情状态
   - 聊天次数
   - 恋人关系（如果有）

4. **每次回复不同** - 由大模型生成，更加自然真实

### 3. 好感度信息加入提示词

**功能描述：**
- 在机器人回复时，将用户的好感度数值添加到提示词中
- 让机器人能够根据好感度调整回复风格

**修改位置：**
- `src/person_info/person_info.py` - `build_relationship()` 方法
- 原来只显示关系等级，现在同时显示好感度数值

**修改前：**
```
张三（关系等级：好友）
```

**修改后：**
```
张三（关系等级：好友，好感度：65.0/100）
```

**提示词中的应用：**
```
**你与 张三 的关系：**
张三（关系等级：好友，好感度：65.0/100）
- 昵称：(ta在qq上的昵称是张三)
- 你还记得有关张三的内容：喜欢打游戏
```

### 4. 主动对话提示词增强

**功能描述：**
- 主动对话时，根据好感度生成更详细的关系提示
- 让机器人根据好感度调整主动对话的亲密程度

**实现位置：**
- `src/proactive_system/proactive_chat_manager.py` - `_build_proactive_chat_prompt()` 方法

**新增提示词内容：**
```
【聊天场景】
- 对象：张三
- 场景：群聊
- 关系等级：好友
- 好感度：65.0/100
- 关系提示：你和张三是好友，关系不错，可以聊得比较轻松自在

【要求】：
9. 根据好感度调整亲密程度：好感度高可以更随意亲昵，好感度低要更礼貌友好
```

**好感度对应的关系提示：**
- 80-100：挚友，关系非常亲密，可以更随意、更亲昵地聊天
- 60-79：好友，关系不错，可以聊得比较轻松自在
- 40-59：熟人，可以聊一些日常话题
- 20-39：刚认识不久，可以主动一些增进了解
- 0-19：还不太熟，可以友好地打个招呼

## 配置说明

### 主动对话配置

在 `config/bot_config.toml` 中：

```toml
[proactive_chat]
enable = true  # 是否启用主动对话
check_interval_minutes = 10  # 检查间隔
min_idle_minutes = 20  # 最小空闲时间
max_daily_proactive = 15  # 每天最多主动发起次数
enable_in_groups = true  # 是否在群聊中启用
enable_in_private = true  # 是否在私聊中启用
```

### 关系系统配置

在 `config/bot_config.toml` 中：

```toml
[relationship]
enable_relationship = true  # 是否启用关系系统
```

## 测试方法

1. **测试关键词触发：**
   ```bash
   python test_intimacy_features.py
   ```

2. **测试主动对话：**
   - 启动 MaiBot
   - 在群聊中保持 20 分钟不说话
   - 观察机器人是否主动 @ 群成员

3. **测试好感度查询：**
   - 在 QQ 中向机器人发送"好感度"
   - 查看返回的信息是否正确

4. **测试提示词中的好感度：**
   - 查看 MaiBot 日志
   - 搜索"你与 XXX 的关系"
   - 确认是否包含好感度数值

## 注意事项

1. **好感度系统需要启用：**
   - 确保 `config/bot_config.toml` 中 `enable_relationship = true`

2. **主动对话需要启用：**
   - 确保 `config/bot_config.toml` 中 `enable = true`（在 `[proactive_chat]` 部分）

3. **插件需要加载：**
   - 好感度查询插件会自动加载
   - 如果没有加载，检查 `plugins/intimacy_query_plugin/` 目录是否存在

4. **数据库需要有数据：**
   - 好感度查询需要用户先与机器人聊天
   - 主动对话选择群成员需要群里有聊天记录
