# 双机器人启动完整指南

## 前提条件检查

在启动前，请确认：

- [x] 已安装Napcat客户端
- [x] 君君的配置文件存在: `config/bot_config.toml`
- [x] 伊伊的配置文件存在: `config/yiyi_bot_config.toml`
- [x] 君君的适配器目录存在: `MaiBot-Napcat-Adapter/`
- [x] 伊伊的适配器目录存在: `MaiBot-Napcat-Adapter-Yiyi/`
- [x] 君君的Napcat配置: `napcat_junjun.json`
- [x] 伊伊的Napcat配置: `napcat_yiyi.json`
- [ ] **重要**: 已配置白名单（否则机器人不会回复）

---

## 方式一: 一键启动（推荐）

### Windows

双击运行或在命令行执行：
```batch
start_all_bots.bat
```

这将自动启动：
1. 君君的Napcat (端口8096)
2. 伊伊的Napcat (端口8097)
3. 君君的适配器
4. 伊伊的适配器
5. MaiBot主程序

### Linux/Mac

```bash
./start_all_bots.sh
```

停止所有服务：
```bash
./stop_all_bots.sh
```

---

## 方式二: 分步启动

### 步骤1: 启动Napcat客户端

#### 君君的Napcat
```bash
napcat --config napcat_junjun.json --qq 2477702109
```

#### 伊伊的Napcat
```bash
napcat --config napcat_yiyi.json --qq 1033245881
```

**注意**:
- 如果是首次登录，需要扫码
- 确保两个Napcat都显示"Connected"

### 步骤2: 启动适配器

#### 君君的适配器
```bash
cd MaiBot-Napcat-Adapter
python main.py
```

#### 伊伊的适配器
打开新终端：
```bash
cd MaiBot-Napcat-Adapter-Yiyi
python main.py
```

**验证**: 适配器应该显示：
```
[INFO] Connected to Napcat at localhost:8096 (或8097)
[INFO] Connected to MaiBot at 127.0.0.1:8000
```

### 步骤3: 启动MaiBot主程序

打开新终端：
```bash
cd MaiBot
python src/main.py
```

**验证**: 应该显示：
```
[INFO] Bot junjun_main started
[INFO] Bot yiyi_bot started
[INFO] Message router initialized with 2 bots
```

## 启动顺序说明

**正确的启动顺序**：
```
1. Napcat客户端 (君君 + 伊伊)
   ↓ 等待5秒
2. 适配器 (君君 + 伊伊)
   ↓ 等待5秒
3. MaiBot主程序
```

**为什么要按顺序**：
- Napcat必须先启动，适配器才能连接
- 适配器必须先连接，MaiBot才能收发消息
- 如果顺序错误，可能导致连接失败

## 验证启动成功

### 1. 检查窗口/终端

应该有5个窗口/终端在运行：
- Napcat-JunJun (君君的QQ连接)
- Napcat-YiYi (伊伊的QQ连接)
- Adapter-JunJun (君君的适配器)
- Adapter-YiYi (伊伊的适配器)
- MaiBot-Main (主程序)

### 2. 检查日志

#### Napcat日志
```
[INFO] WebSocket server started on 0.0.0.0:8096 (或8097)
[INFO] Connected to QQ: 2477702109 (或1033245881)
```

#### 适配器日志
```
[INFO] Connected to Napcat at localhost:8096 (或8097)
[INFO] Connected to MaiBot at 127.0.0.1:8000
[INFO] Heartbeat received
```

#### MaiBot日志
```
[INFO] Bot junjun_main started
[INFO] Bot yiyi_bot started
[INFO] Message router initialized with 2 bots
[INFO] WebUI server started on http://0.0.0.0:8001
```

### 3. 访问管理界面

打开浏览器访问：
```
http://localhost:8001/bot-management.html
```

应该看到：
- 君君 (junjun_main) - 状态: 运行中
- 伊伊 (yiyi_bot) - 状态: 运行中

### 4. 发送测试消息

向君君或伊伊发送消息，检查是否回复。

## 常见问题排查

### Q1: Napcat启动失败

**症状**: Napcat窗口闪退或报错

**解决**:
1. 检查Napcat是否正确安装
2. 检查配置文件是否存在
3. 检查端口是否被占用
4. 查看Napcat日志文件

### Q2: 适配器连接失败

**症状**: 适配器显示"Connection refused"

**解决**:
1. 确认Napcat已经启动
2. 检查端口配置是否正确
3. 检查防火墙设置
4. 确认config.toml中的端口与Napcat配置一致

### Q3: MaiBot无法识别机器人

**症状**: MaiBot日志中只显示一个机器人启动

**解决**:
1. 检查多机器人模式是否启用
2. 检查配置文件路径是否正确
3. 查看MaiBot日志中的错误信息

### Q4: 机器人不回复消息

**症状**: 发送消息后没有回复

**解决**:
1. **检查白名单配置**（最常见原因）
2. 检查适配器是否正常连接
3. 检查MaiBot是否收到消息（查看日志）
4. 检查机器人的talk_value设置

### Q5: 端口冲突

**症状**: 启动时提示端口已被占用

**解决**:
```bash
# Windows - 查看端口占用
netstat -ano | findstr "8096"
netstat -ano | findstr "8097"

# Linux/Mac - 查看端口占用
lsof -i :8096
lsof -i :8097

# 修改配置文件中的端口号
```

## 白名单配置（重要）

### 君君的白名单

编辑 `MaiBot-Napcat-Adapter/config.toml`：
```toml
[chat]
group_list = [你的测试群号]
private_list = [你的QQ号]
```

### 伊伊的白名单

编辑 `MaiBot-Napcat-Adapter-Yiyi/config.toml`：
```toml
[chat]
group_list = [你的测试群号]
private_list = [你的QQ号]
```

**注意**:
- 如果不配置白名单，机器人将无法回复任何消息
- 可以为两个机器人配置不同的白名单
- 修改白名单后需要重启适配器

## 停止服务

### Windows

关闭所有打开的窗口，或按Ctrl+C停止每个进程。

### Linux/Mac

使用停止脚本：
```bash
./stop_all_bots.sh
```

或手动停止：
```bash
# 查找进程
ps aux | grep napcat
ps aux | grep "main.py"

# 停止进程
kill <PID>
```

## 性能优化建议

### 1. 调整日志级别

如果日志太多，可以调整为WARNING：

`config.toml`:
```toml
[debug]
level = "WARNING"
```

### 2. 减少心跳频率

如果网络稳定，可以增加心跳间隔：

`config.toml`:
```toml
[napcat_server]
heartbeat_interval = 60  # 从30秒改为60秒
```

### 3. 使用后台运行

Linux/Mac可以使用nohup后台运行：
```bash
nohup ./start_all_bots.sh > maibot.log 2>&1 &
```

## 开机自启动（可选）

### Windows

1. 创建快捷方式到 `start_all_bots.bat`
2. 将快捷方式放到启动文件夹：
   ```
   %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup
   ```

### Linux (systemd)

创建服务文件 `/etc/systemd/system/maibot.service`：
```ini
[Unit]
Description=MaiBot Multi-Bot System
After=network.target

[Service]
Type=forking
User=your_username
WorkingDirectory=/path/to/MaiM-with-u
ExecStart=/path/to/MaiM-with-u/start_all_bots.sh
ExecStop=/path/to/MaiM-with-u/stop_all_bots.sh

[Install]
WantedBy=multi-user.target
```

启用服务：
```bash
sudo systemctl enable maibot
sudo systemctl start maibot
```
